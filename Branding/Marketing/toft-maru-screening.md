# Toft & Maru — Preliminary Screening Results

_Prepared: 2026-05-19. Preliminary screening only, not legal advice. A proper attorney-led trademark search is still required before any commercial launch under any name._

## Verdicts

**Toft — passes preliminary screen.** First name in four rounds (Stead, Mantel, Folio, Toft) to come back without a high-priority collision in the consumer household-management space. Surfaced "Toft" uses are all in different categories or different markets: Danish B2B software development consultancies, a B2B field activity management app for utilities crews, an open-source infrastructure testing library on GitHub, and an industrial-instrument software tool. None of these are consumer apps, none are in the household-management category, and none have brand presence likely to contest a US consumer app launch.

**Maru — does not pass.** Multiple active software brands already use the name across adjacent SaaS categories. Combined with the cultural-appropriation concern flagged in the v2 doc, the case for Maru collapses.

## Toft — evidence

**TOFT Software (toft-software.dk).** Danish app development and .NET consulting firm. Not a consumer brand, geographically focused on Denmark, B2B services. No app-store presence, no consumer-app namespace conflict.

**TOFT Solutions (toftsolutions.com).** IT solutions consultancy. B2B services. No category collision.

**TOFT — Google Play app (com.threeforcom.toft).** A field activity management app published by "threeforcom", updated February 2025. **This is the closest live collision** — it's on Google Play under the name TOFT. However, it serves utilities/field-services teams (a B2B field-management category), not consumer households. App Store and Play Store policies generally allow same-name apps when they're in materially different categories and serve clearly different users. Should be confirmed at filing, but is unlikely to block a consumer household app.

**Toft (open-source infrastructure testing library — github.com/exceedhl/toft).** Code library for chef/shell-script testing with lxc. Not a brand, a repo. No collision.

**ToFTool (Endress + Hauser industrial software).** Industrial instrument software using time-of-flight measurement. Different spelling capitalization, different category, B2B/industrial.

**TOff (Power Saver & Scheduler).** Different name (TOff, not Toft). No collision.

**Multiple GitHub profiles named "Toft"** (e.g. `toftt`, `t-toft`). "Toft" is a Scandinavian surname (Danish, Norwegian, Swedish), so there's incidental personal-profile noise across professional networks. None of these are brand competitors.

## What Toft has going for it

- **Real meaning intact.** Old English / Old Norse for a homestead with its surrounding plot of land. Semantically perfect for a household-management product. Better than a pure coinage because the meaning supports the positioning rather than needing to be invented.
- **Ultra-short.** Four letters, one syllable. Sets crisply in Fraunces in the small wordmark slot. Easy to say.
- **Defensible namespace.** No active consumer brand uses Toft; B2B/industrial uses are far enough away to coexist without conflict.
- **Domain pattern is likely open.** `toft.app`, `toft.family`, `toft.house` all seem unclaimed by major brands based on search results. (Not yet verified with WHOIS — recommend that as the next step.)
- **Trademark space appears clean.** No registered consumer-app marks for TOFT surfaced in search; an attorney USPTO pass is still required, but the preliminary signal is favorable.

## What Toft has working against it

- **Recognizability is low.** Most users won't know the word. Functions almost like a coinage for brand-building purposes — the tagline and brand voice will need to do explanatory work in early product surfaces (login screen, App Store description, install prompt).
- **Pronunciation drift possible.** Correct pronunciation is "toft" (rhymes with "soft"), but some users will say it differently on first encounter. Worth front-loading pronunciation guidance in onboarding ("Toft. Rhymes with soft.") or letting users discover it through hearing it spoken in marketing.
- **Scandinavian-surname namespace noise.** Many real people named Toft exist (researchers, developers, professionals). Search for "Toft" surfaces them. Not a brand-collision issue, but the SEO surface has noise. Mitigated by adding qualifiers in marketing ("Toft, the home app" / "Toft.app").
- **Closest collision is on Google Play.** The TOFT field-management app on Android is the one place where direct same-name same-store appearance is possible. Likely acceptable in practice (different category, different audience), but worth verifying with the eventual store submission.

## Maru — evidence

**Maru OS (maruos.com).** Active open-source operating system project that "makes personal computing more context-aware". Distinctive software brand. Different category from home-pm but same broader "consumer software named Maru" namespace.

**Maru/HUB (marugroup.net).** Market research and insights technology platform. Active B2B SaaS, multiple SourceForge / Capterra / Slashdot listings. Established brand with vendor presence.

**Maru AI (Atlassian Marketplace).** Active AI assistant for software engineering teams, vendor ID 4230505136 on the Atlassian Marketplace. Direct software-brand collision.

**Maru Group (themarugroup.medium.com).** "Software + Advisory Services" consultancy. Active brand presence.

**@getmaru (X/Twitter).** Live handle. Unclear product but the URL pattern recommended in the v2 doc (`getmaru.com`) is in use.

**Cultural surface.** Maru is also one of the most famous cats on the internet — a Japanese Scottish-Fold-style cat with millions of YouTube subscribers since the late 2000s. For most English-speaking users, the dominant "Maru" association is the cat, not any of the software products.

**Cultural-appropriation concern.** Even setting aside the namespace collision, a US-based household app using a Japanese word is a non-trivial choice. It's not automatically problematic — many products do this thoughtfully — but it requires intentional cultural attention: pronunciation, possible Japanese-language marketing, awareness that some Japanese-American users may view it as decorative use of a non-English word. This is a real conversation, not a screening question.

## Net recommendation

**Treat Toft as the working name and proceed.** It's the first name in four rounds to clear preliminary screening. The remaining caveats (low recognizability, surname namespace noise, one Google Play category-adjacent app) are all manageable with brand-building work; none are disqualifying.

The right next steps, in sequence:

1. **Domain WHOIS check** on `toft.app`, `toft.family`, `toft.house`, `toft.com`. Confirm availability before committing.
2. **Social handle check** on `@toft`, `@toftapp`, `@thetoft` across X, Instagram, TikTok, Threads, Bluesky, GitHub.
3. **Attorney-led USPTO TESS search** (and UKIPO/EUIPO if international reach is in scope). This is the bar for commercial launch.
4. **Hold off on the codebase rename** until step 3 clears. Until then, refer to home-pm as "The Board (working title)" externally and "Toft (proposed)" internally.

If any of the above turns up a blocker, fall back to running the rest of the v2 list (Hometide, Kintly, Hometable, Stoop, Eaves, Steading, Around Here, Common House).

## Caveat

This screen relied on public web evidence only. It did not include USPTO TESS, UKIPO, EUIPO, or common-law searches by a trademark attorney. Toft passing this screen is necessary but not sufficient for commercial use — the attorney pass is still required.
